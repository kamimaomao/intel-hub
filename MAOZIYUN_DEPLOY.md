# 帽子云部署配置

## 前端静态部署配置

- 应用类型：静态应用
- 仓库分支：部署当前项目所在分支
- 安装命令：`npm install`
- 构建命令：`npm run build`
- 输出目录：`dist`
- Node.js：20 或更高版本
- 可选环境变量：`VITE_API_BASE=https://你的后端域名`

## 真后端部署说明

项目已经包含 Node/Express 后端：

- 启动命令：`npm start`
- 默认端口：`8787`，可通过 `PORT` 环境变量覆盖
- 数据目录：`data/`，可通过 `DATA_DIR` 环境变量覆盖
- API：`/api/login`、`/api/items`、`/api/sources`

帽子云官方文档说明它主要提供静态网站/前端服务，暂不支持 Next.js/Nuxt.js 这类需要服务器运行时的全栈应用。因此要保留真后端，有两种部署方式：

1. 帽子云部署前端静态产物，使用反向代理把 `/api/*` 转发到外部 Node 服务。
2. 把整个项目部署到支持 Node 服务的主机，再由帽子云只做前端或域名入口。

如果后端域名固定，也可以在帽子云环境变量里设置 `VITE_API_BASE`，前端会直接请求该地址。

## 部署步骤

1. 把当前项目推送到 GitHub 仓库。
2. 在帽子云中新建静态应用。
3. 授权并选择 GitHub 仓库与分支。
4. 填入上面的构建配置。
5. 创建应用，等待构建完成后访问帽子云分配的域名。

## 当前限制

当前后端使用服务端 JSON 文件作为最小持久化。它比浏览器本地存储更接近真实后台，但还不是最终生产数据库。多人长期使用建议换成 Postgres/Supabase，并补管理员鉴权、抓取任务和备份。
