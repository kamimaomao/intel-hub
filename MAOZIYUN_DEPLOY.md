# 帽子云部署配置

## 重要结论

这版已经接入原站真实数据，不能只按“静态网站”部署。静态网站不会运行 `server/index.mjs`，因此后端无法登录原站，也无法读取真实文章。

## Node 服务部署配置

- 应用类型：Node / Web Service / 服务端应用
- 仓库分支：`main`
- 安装命令：`npm install`
- 构建命令：`npm run build`
- 启动命令：`npm start`
- Node.js：20.18.1 或更高版本
- 端口：使用平台自动注入的 `PORT`，本项目会自动读取

必填环境变量：

- `XIANJIAN_USERNAME`：原站账号
- `XIANJIAN_PASSWORD`：原站密码

可选环境变量：

- `DATA_DIR`：公众号源数据目录，默认 `data/`

不配置原站账号密码时，真实文章接口会返回认证错误。前台已经没有登录页，原站登录只由后端环境变量完成。

## 如果帽子云只能创建静态网站

保留真实数据需要两段式部署：

1. 在支持 Node 的平台部署本项目后端，启动命令仍然是 `npm start`。
2. 帽子云部署前端静态产物，并设置环境变量 `VITE_API_BASE=https://你的后端域名`。

## 部署步骤

1. 把当前项目推送到 GitHub 仓库。
2. 在帽子云中新建 Node 服务端应用。
3. 授权并选择 GitHub 仓库 `kamimaomao/intel-hub` 与 `main` 分支。
4. 填入上面的安装、构建、启动命令。
5. 配置原站账号密码环境变量。
6. 创建应用，等待构建完成后访问帽子云分配的域名。

## 当前限制

当前后端实时代理原站页面并解析 HTML。它已经是真实数据源，但依赖原站页面结构和登录态；如果原站改版，解析规则需要同步调整。
